# MANU 投资报告可视化

将曼联（MANU）投资研究报告从人读的文本报告，转为结构化数据资产，并通过一个基础 webapp 进行可视化展示。

## 项目背景

原始投资报告是一份传统的文本型研究报告（PDF）。本项目的目标是探索"研究报告结构化 → 数据资产 → 可视化展示"这一工作流，为后续研究成果的多形态呈现打基础。

**当前阶段目标**：跑通"数据 → 展示"的完整链路，验证结构化方案的可行性，尚非最终视觉方案。

## 技术栈

纯 HTML + CSS + JavaScript，零构建工具，无框架依赖。

选择这套技术栈是有意为之：本阶段追求的是简单、可直接运行、无技术债的实现，为后续视觉迭代留出充分空间。

## 数据结构

- `manu_report.schema.json`：数据字段定义（JSON Schema），描述报告中各类信息（投资评级、财务历史、估值模型、SWOT 分析等）的结构与类型约束。
- `manu_report.data.json`：基于上述 schema 抽取的实际数据实例。

数据抽取遵循以下原则：
- 不编造原文未明确给出的数值，模糊表述保留原有的不确定性
- 每个金额字段独立标注货币单位（原报告存在英镑/欧元混用）
- 报告原文本身存在的数据口径矛盾（如企业价值计算、估值倍数等 5 处）如实标注在页面「数据说明」区域，未做修正

## 本地运行

由于代码通过 `fetch()` 读取本地 JSON 文件，直接双击打开 `index.html` 可能因浏览器安全策略无法加载数据，建议启动一个简单的本地服务器：

```bash
# 进入项目目录后执行
python3 -m http.server 8765
```

然后在浏览器访问 `http://127.0.0.1:8765`。

## 页面结构

1. 封面（评级、目标价、涨幅）
2. 核心论点与风险提示
3. 财务表现（五年趋势）
4. 估值分析（FCFF 模型、同业对比）
5. 竞争格局（SWOT）
6. 投资建议
7. 数据说明（原文数据矛盾的透明标注）
